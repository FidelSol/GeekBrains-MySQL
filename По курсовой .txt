База данных взята с проекта django - Система генерации отчетов, а именно: с раздела пользователя.
Замысел в том, чтобы администратор приложения формировал типы отчетов - шаблоны, которые определяют вид и количество параметров запроса, 
а пользователи по доступным им типам отчета создавали запрос, заполняя его параметры согласно типам отчета. Отправив запрос на другой сервер, приходит результат.
В базе данных созданы следующие таблицы:
report_customuser - Пользователь;
report_customuser_levels - Доступы по типам отчета - промежуточная таблица связи многие-ко-многим с таблицей Тип отчета;
report_perm - Уровень доступа (в приложении три уровня - администратор, может делать запрос и смотреть результат, может только смотреть результат;
report_role - Должности;
report_typereport - Типы отчета;
report_targetfile - Файловые параметры Типа отчета - many-to-one;
report_targetstring - Текстовые параметры Типа отчета - many-to-one;
report_targettime - Временные параметры Типа отчета - many-to-one;
report_query - Запрос;
report_textparameter - Текстовые параметры Запроса - many-to-one;
report_timeparameter - Временные параметры Запроса - many-to-one;
report_fileparameter - Файловые параметры Запроса - many-to-one;
report_test - Тестовые таблицы (не удалил еще со времен, когда разрабатывал проект - будет удалена)
report_result - Результаты;
report_resultcounter - Счетчики - если пользователь скачал результат - создается сущность в счетчике - это для отображения непрсомотренных результатов в интерфейсе;

Остальные таблицы идут по-умолчанию в проекте django, я их могу убрать при необходимости. Это таблицы стандартного пользователя auth_user, групп groups, прав, миграций и content_types,
 которые я не использовал, потому что преписывал бэкенд по доступам и пользователю по-своему. Они всегда создаются стандартно.
